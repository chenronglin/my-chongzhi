# V1 后台 UI 设计说明

本文基于 [FRONT_BUS.md](FRONT_BUS.md) 中定义的 V1 前端业务范围，继续细化为 UI 设计层面的页面说明。

本文重点回答三个问题：

- 每个模块有哪些页面
- 每个页面应包含哪些主要屏幕元素
- 每个页面承担什么业务功能

本文不讨论接口、组件库、前端技术方案等实现细节。

## 1. 整体产品形态

V1 产品形态是一套面向内部运营、配置、风控、财务和技术支持团队的管理后台。

整体界面建议采用“顶部全局栏 + 左侧主导航 + 右侧内容区”的后台结构，保证信息密度、检索效率和跨模块跳转效率。

### 1.1 一级导航建议

- 工作台
- 登录与权限
- 渠道管理
- 商品中心
- 风控管理
- 订单中心
- 支付管理
- 供应商履约
- 结算与账务
- 通知管理
- Worker 任务

### 1.2 全局公共区域

所有后台页面建议统一具备以下全局元素：

- 顶部栏：系统名称、全局搜索入口、消息提醒入口、当前用户信息、退出登录入口
- 左侧导航：一级模块导航、当前模块高亮、可折叠导航区
- 面包屑：展示当前页面路径，便于快速定位
- 页面标题区：页面名称、页面说明、主要操作按钮
- 状态提示区：成功、失败、风险、待处理等重要业务提示

### 1.3 全局通用设计原则

- 列表页优先强调筛选、状态识别和快速跳转
- 详情页优先强调业务全貌、状态进展和关联信息
- 关键异常操作应使用确认弹窗，避免误操作
- 风险、失败、待处理状态应有明显颜色区分和提示文案
- 订单、支付、通知、任务等链路型信息应尽量使用时间线或步骤流展示

## 2. 工作台

工作台不是交易入口，而是管理后台首页，用于帮助业务人员快速掌握系统运行情况。

### 2.1 工作台首页

页面定位：
作为运营、风控、客服、财务等角色进入系统后的默认首页，展示当前系统最重要的业务总览和待处理事项。

主要屏幕元素：

- 顶部欢迎区：当前日期、当前登录人、快捷入口
- 核心数据卡片区：订单总量、成功订单数、退款订单数、异常订单数、通知失败数、死信任务数
- 订单趋势区：最近一段时间订单状态变化趋势
- 待处理事项区：待重试通知、死信任务、异常订单、人工复核订单提示
- 快捷操作区：新增渠道、新增商品、查看订单、查看任务、查看通知
- 最近操作区：显示最近访问的页面或最近处理的业务对象

功能描述：

- 帮助用户快速感知当前业务是否正常运行
- 为异常处理提供直接跳转入口
- 为高频操作提供快捷进入方式

## 3. 登录与权限

### 3.1 登录页

页面定位：
后台系统统一登录入口，用于管理员进入系统。

主要屏幕元素：

- 品牌区：系统名称、产品定位说明
- 登录表单区：账号输入框、密码输入框、登录按钮
- 登录状态提示区：错误提示、账号异常提示、密码错误提示
- 辅助信息区：安全提示、登录说明

功能描述：

- 支持管理员输入账号密码完成登录
- 在登录失败时明确反馈失败原因
- 登录成功后进入工作台或上次访问页面

### 3.2 用户列表页

页面定位：
用于查看和管理后台用户。

主要屏幕元素：

- 页面标题区：用户管理、说明文案、新增用户按钮
- 筛选区：用户名、账号状态、角色、创建时间
- 用户表格区：用户名、登录账号、角色、状态、创建时间、最近登录时间、操作列
- 操作区：查看详情、分配角色、新增用户
- 分页区：页码、总数、每页条数切换

功能描述：

- 查看当前后台用户信息
- 按角色或状态筛选目标用户
- 发起新增用户操作
- 为后续权限分配和人员管理提供入口

### 3.3 新增用户页或弹窗

页面定位：
用于录入新的后台用户信息。

主要屏幕元素：

- 基础信息表单：姓名、登录账号、初始密码、手机号或邮箱
- 角色选择区：单选或多选角色
- 状态设置区：是否启用
- 操作按钮区：保存、取消

功能描述：

- 创建新的后台管理账号
- 为新用户设置初始角色和状态

### 3.4 角色列表页

页面定位：
用于查看系统内已有角色。

主要屏幕元素：

- 页面标题区：角色管理、新增角色按钮
- 筛选区：角色名称、创建时间
- 角色表格区：角色名称、角色说明、关联用户数、创建时间、操作列

功能描述：

- 查看已有角色及其基本说明
- 识别不同角色在系统中的使用情况
- 发起新增角色操作

### 3.5 新增角色页或弹窗

页面定位：
用于创建基础角色。

主要屏幕元素：

- 角色信息表单：角色名称、角色说明
- 权限说明区：当前角色定位说明
- 操作按钮区：保存、取消

功能描述：

- 创建新的后台角色
- 为后续用户分配角色提供基础

## 4. 渠道管理

### 4.1 渠道列表页

页面定位：
用于查看所有渠道主体及其接入状态，是渠道管理的总入口。

主要屏幕元素：

- 页面标题区：渠道管理、新增渠道按钮
- 筛选区：渠道名称、渠道编码、状态、是否配置回调、创建时间
- 渠道表格区：渠道名称、渠道编码、状态、授权商品数、回调状态、最近更新人、更新时间、操作列
- 操作列：查看详情、进入配置

功能描述：

- 快速查看渠道是否完成接入准备
- 快速定位未配置回调、未授权商品等问题渠道
- 进入渠道详情完成后续配置

### 4.2 新增渠道页

页面定位：
用于创建新的渠道主体。

主要屏幕元素：

- 基础信息表单：渠道名称、渠道编码、联系人、联系方式、说明
- 状态设置区：是否启用
- 提交按钮区：保存、取消

功能描述：

- 创建渠道基础档案
- 为后续配置接入凭证、商品授权、价格和限额建立主体

### 4.3 渠道详情页

页面定位：
用于展示和维护单个渠道的全部业务配置，是渠道模块的核心页面。

主要屏幕元素：

- 详情头部区：渠道名称、渠道编码、状态标签、快捷操作按钮
- 摘要卡片区：授权商品数、已配置价格数、限额配置状态、回调配置状态
- 标签页导航：基础信息、接入凭证、商品授权、价格配置、限额配置、回调配置

功能描述：

- 汇总展示渠道的整体配置完成度
- 以标签页形式承载不同配置主题
- 让运营人员在一个页面内完成渠道全量配置

### 4.4 渠道详情-基础信息页签

页面定位：
展示渠道主体基本资料。

主要屏幕元素：

- 基础信息卡片：渠道名称、渠道编码、联系人、联系方式、创建时间、备注
- 状态信息卡片：启用状态、最近更新时间

功能描述：

- 查看渠道身份信息和基础状态
- 作为核对渠道资料的只读页面

### 4.5 渠道详情-接入凭证页签

页面定位：
用于查看和维护渠道接入开放接口所需的凭证信息。

主要屏幕元素：

- 凭证信息卡片：AccessKey、SecretKey 展示区
- 安全提示区：凭证用途说明、保密提示
- 操作按钮区：更新凭证、复制凭证

功能描述：

- 支撑渠道开放接口接入
- 让运营或技术支持人员维护渠道签名接入信息

### 4.6 渠道详情-商品授权页签

页面定位：
用于配置某个渠道可售的商品与 SKU 范围。

主要屏幕元素：

- 授权筛选区：商品分类、商品名称、SKU 状态
- 可选商品表格区：商品名称、SKU、上架状态、授权状态
- 已授权清单区：已授权商品和 SKU 列表
- 操作按钮区：添加授权、移除授权

功能描述：

- 设定渠道能够销售哪些商品
- 让商品可售范围与渠道接入策略保持一致

### 4.7 渠道详情-价格配置页签

页面定位：
用于配置渠道在不同 SKU 下的价格口径。

主要屏幕元素：

- SKU 价格表格：商品名称、SKU、标准价、渠道价、更新时间、操作列
- 筛选区：商品名称、SKU、是否已配置价格
- 编辑入口：设置价格、更新价格

功能描述：

- 维护渠道维度的 SKU 价格
- 帮助运营人员快速识别哪些 SKU 尚未配置价格

### 4.8 渠道详情-限额配置页签

页面定位：
用于配置渠道的交易限额策略。

主要屏幕元素：

- 限额配置表单：单笔限额、日限额、月限额、QPS 限额
- 状态提示区：当前配置是否完整
- 说明区：限额口径说明
- 操作按钮区：保存、重置

功能描述：

- 维护渠道交易风险边界
- 为下单前的限额校验提供业务配置基础

### 4.9 渠道详情-回调配置页签

页面定位：
用于维护渠道订单通知所需的回调信息。

主要屏幕元素：

- 回调信息表单：回调地址、回调签名秘钥、通知说明
- 配置状态卡片：是否已配置、最近更新时间
- 安全提示区：签名验证说明

功能描述：

- 配置订单成功或退款成功后的通知回调基础信息
- 帮助业务人员确认通知链路是否具备执行条件

## 5. 商品中心

### 5.1 商品分类列表页

页面定位：
用于维护商品分类体系。

主要屏幕元素：

- 页面标题区：商品分类、新增分类按钮
- 筛选区：分类名称、状态
- 分类表格区：分类名称、分类编码、状态、商品数量、创建时间、操作列

功能描述：

- 管理商品所属分类
- 为商品检索和授权配置提供分类基础

### 5.2 新增商品分类页或弹窗

页面定位：
用于新增商品分类。

主要屏幕元素：

- 分类表单：分类名称、分类编码、分类说明、状态
- 操作按钮区：保存、取消

功能描述：

- 创建商品分类
- 规范商品归类结构

### 5.3 商品列表页

页面定位：
用于查看所有商品及其可售状态。

主要屏幕元素：

- 页面标题区：商品中心、新增商品按钮
- 筛选区：商品名称、商品分类、可售状态、创建时间
- 商品表格区：商品名称、商品编码、分类、可售状态、SKU 数量、供应商映射状态、更新时间、操作列
- 操作列：查看详情、新增 SKU

功能描述：

- 统一查看商品主档信息
- 快速识别不可售商品和未完成供应商映射的商品

### 5.4 新增商品页

页面定位：
用于录入新的商品主档。

主要屏幕元素：

- 商品表单：商品名称、商品编码、所属分类、商品说明、可售状态
- 操作按钮区：保存、取消

功能描述：

- 创建新的商品
- 为后续 SKU 管理和渠道授权提供商品主体

### 5.5 商品详情页

页面定位：
用于查看商品基本信息和下属 SKU，是商品模块的核心页面。

主要屏幕元素：

- 详情头部区：商品名称、商品编码、状态标签、快捷操作按钮
- 商品信息卡片：分类、说明、创建时间、更新时间
- 摘要卡片区：SKU 数量、已映射供应商 SKU 数量
- 标签页导航：基础信息、SKU 管理、供应商映射

功能描述：

- 汇总展示商品基础资料和 SKU 完整度
- 将商品基本管理与 SKU 层配置集中处理

### 5.6 商品详情-SKU 管理页签

页面定位：
用于管理某个商品下的所有 SKU。

主要屏幕元素：

- SKU 工具栏：新增 SKU 按钮、状态筛选
- SKU 表格区：SKU 名称、SKU 编码、上架状态、映射状态、更新时间、操作列
- 操作入口：查看 SKU、维护状态、进入映射配置

功能描述：

- 查看和维护商品下的 SKU 结构
- 快速发现未上架或未完成映射的 SKU

### 5.7 新增 SKU 页或弹窗

页面定位：
用于为商品新增 SKU。

主要屏幕元素：

- SKU 表单：SKU 名称、SKU 编码、上架状态、业务说明
- 操作按钮区：保存、取消

功能描述：

- 创建商品的具体销售单元
- 为后续渠道授权和供应商履约建立最小业务单位

### 5.8 商品详情-供应商映射页签

页面定位：
用于配置 SKU 与供应商能力之间的静态映射关系。

主要屏幕元素：

- 映射表格区：SKU、候选供应商、供应商 SKU、映射状态、更新时间、操作列
- 筛选区：SKU、供应商、映射状态
- 编辑入口：新增映射、更新映射

功能描述：

- 维护每个 SKU 对应的供应商履约关系
- 保证订单创建后能够找到有效的履约路径

## 6. 风控管理

### 6.1 风控规则列表页

页面定位：
用于查看系统内全部风控规则。

主要屏幕元素：

- 页面标题区：风控管理、新增规则按钮
- 筛选区：规则类型、命中对象、决策结果、状态、创建时间
- 规则表格区：规则名称、规则类型、命中对象、决策结果、状态、更新时间、操作列
- 状态标识区：通过、人工复核、拒绝的视觉标签

功能描述：

- 统一查看当前启用的风控规则
- 帮助风控人员理解 V1 的风险判断依据

### 6.2 新增风控规则页

页面定位：
用于创建新的风控规则。

主要屏幕元素：

- 规则基础表单：规则名称、规则类型、命中对象、状态
- 决策设置区：通过、人工复核、拒绝
- 条件配置区：黑名单值、白名单值、金额阈值等业务条件
- 规则说明区：适用范围与备注
- 操作按钮区：保存、取消

功能描述：

- 新增黑名单、白名单、金额阈值类规则
- 明确命中规则后的业务处理结果

### 6.3 风控规则详情页

页面定位：
用于查看单条风控规则的详细配置。

主要屏幕元素：

- 规则概览卡片：规则名称、规则类型、状态、决策结果
- 规则条件卡片：命中条件、适用对象、阈值说明
- 变更信息区：创建人、创建时间、更新时间

功能描述：

- 便于风控人员核对规则配置
- 支撑问题排查时快速查看规则定义

## 7. 订单中心

### 7.1 订单列表页

页面定位：
订单中心主页面，用于查看所有订单并进行快速筛选和跳转。

主要屏幕元素：

- 页面标题区：订单中心
- 高级筛选区：订单号、渠道订单号、渠道、商品、主状态、支付状态、供应商状态、通知状态、风控状态、创建时间
- 快捷筛选标签：异常订单、待支付、待提交供应商、处理中、已成功、退款中、已退款、已关闭
- 订单表格区：订单号、渠道名称、商品名称、SKU、订单金额、主状态、支付状态、供应商状态、通知状态、创建时间、操作列
- 操作列：查看详情、追加备注、标记异常、人工关闭

功能描述：

- 以订单为核心查看全链路业务状态
- 快速定位异常订单和关键状态订单
- 进入详情页查看完整业务过程

### 7.2 订单详情页

页面定位：
用于查看单笔订单的完整业务信息，是整个后台最关键的业务详情页。

主要屏幕元素：

- 订单头部区：订单号、主状态标签、异常标记、快捷操作按钮
- 核心摘要卡片：订单金额、渠道名称、商品名称、SKU、创建时间
- 状态进展区：支付、供应商、通知、退款等步骤流或时间线
- 基础信息区：渠道订单号、回调信息、风险结论、扩展说明
- 标签页导航：订单概览、业务快照、事件轨迹、支付信息、履约信息、通知信息、账务信息、备注记录

功能描述：

- 集中展示订单从创建到完成的全部业务环节
- 作为客服、运营、技术支持排查问题的核心页面
- 承载人工关闭、标记异常、追加备注等关键动作

### 7.3 订单详情-订单概览页签

页面定位：

## 14. V1 设计重点结论

从 UI 设计角度看，V1 后台最重要的不是页面数量，而是三类能力是否被清晰承接：

- 配置类能力：渠道、商品、供应商、风控、分润等基础配置
- 观测类能力：订单、支付、履约、账务、通知、任务的状态查看
- 处置类能力：异常标记、人工关闭、通知重试、任务重试、任务取消

因此，整套后台的设计重点应放在以下三个方向：

- 列表筛选效率要高，便于快速查到业务对象
- 详情信息要完整，便于还原业务链路
- 异常入口要明显，便于快速补偿和处理

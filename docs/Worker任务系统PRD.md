## 2. 模块背景与定位

### 2.1 模块背景

支付回调之后的履约提交、供应商轮询、通知重试、对账与补偿都不适合放在同步请求链路中，需要独立异步执行系统承接。

### 2.2 一句话定位

Worker 任务系统是平台统一的异步任务执行与补偿服务，负责任务调度、重试、退避、死信与执行审计，但不承载业务规则定义。

### 2.3 模块价值

1. 解耦主链路与耗时任务。
2. 统一异步任务的重试与补偿能力。
3. 统一任务执行日志与死信处理。

## 3. 职责边界

### 3.1 Worker 任务系统负责什么

1. 异步任务入队与调度。
2. 任务重试、退避与去重。
3. 死信与人工重放。
4. 任务执行日志与运行监控。

### 3.2 Worker 任务系统不负责什么

1. 不定义订单、支付、供应商、通知的业务规则。
2. 不直接修改业务主表事实。
3. 不承载人工审批逻辑。

## 4. 设计原则

### 4.1 任务与业务规则分离原则

Worker 只负责“何时执行”和“如何重试”，不负责“是否应该执行”的业务判断。

### 4.2 幂等执行原则

任务可重复执行，但业务结果必须可幂等。

### 4.3 失败可恢复原则

失败任务应进入重试或死信，不允许静默丢失。

## 5. 模块目标与成功标准

### 5.1 业务目标

- 承接所有异步耗时任务。
- 提供稳定重试与补偿能力。

### 5.2 技术目标

- 支持不同任务类型与退避策略。
- 支持高可用与可观测。

### 5.3 成功标准

- 关键异步任务不因单次失败丢失。
- 死信任务可定位并人工重放。

## 6. 角色与使用场景

| 角色 | 使用场景 |
|---|---|
| 订单服务 | 投递履约、通知、超时关闭等任务 |
| 供应商服务 | 被调度执行查询与补偿 |
| 通知服务 | 被调度执行重试 |
| 财务 / 运维 | 查看任务日志、重放死信 |

- `NEW`
- `READY`
- `RUNNING`
- `SUCCESS`
- `FAIL`
- `RETRY_WAIT`
- `DEAD_LETTER`
- `CANCELED`

## 9. 核心流程设计

### 9.1 入队流程

1. 业务服务创建任务请求。
2. Worker 任务系统按 `jobType + businessKey` 做去重。
3. 任务进入待执行队列。

### 9.2 执行与重试流程

1. Worker 拉取就绪任务。
2. 调用对应业务执行器。
3. 失败则按退避策略进入重试。
4. 超过阈值则进入死信。

## 10. 功能需求明细

### 10.1 任务类型

V1 建议支持：

- 供应商提交
- 供应商查询
- 退款补偿
- 通知重试
- 订单超时关闭
- 结算批处理
- 对账批处理

### 10.2 调度能力

- 支持立即执行、延迟执行、定时执行。

### 10.3 死信处理

- 支持查看死信详情。
- 支持人工重放、取消。

## 11. 核心规则

1. 任务必须具备业务键。
2. 任务执行日志必须保留。
3. 任务系统不直接决定业务终态。

## 12. API 设计

### 12.1 后台管理 API

- `GET /admin/jobs`
- `GET /admin/jobs/{jobId}`
- `POST /admin/jobs/{jobId}/retry`
- `POST /admin/jobs/{jobId}/cancel`
- `GET /admin/jobs/dead-letters`

### 12.2 内部服务 API

- `POST /internal/jobs/enqueue`
- `POST /internal/jobs/schedule`

## 13. 数据模型设计

| 表名 | 用途 |
|---|---|
| `worker_jobs` | 任务主表 |
| `worker_job_attempts` | 执行尝试记录 |
| `worker_dead_letters` | 死信记录 |

## 14. 跨服务协作

### 14.1 与订单服务

- 承接履约提交、通知、超时关闭等任务。

### 14.2 与供应商服务

- 调度查询、退款补偿等动作。

### 14.3 与通知服务

- 调度重试与死信重放。

### 14.4 与结算与账务服务

- 调度批量结算与对账任务。

## 15. 异常处理

### 15.1 任务重复创建

- 依据业务键去重。

### 15.2 执行器异常

- 记录错误并按策略重试。

### 15.3 长期失败

- 转入死信。

## 16. 权限、审计与非功能要求

### 16.1 权限要求

- 人工重试、死信重放、取消任务需重点审计。

### 16.2 非功能要求

- 支持水平扩展。
- 任务日志与执行记录可检索。

## 17. 风险点与控制策略

| 风险 | 影响 | 控制策略 |
|---|---|---|
| 任务丢失 | 异步链路断裂 | 持久化与重试 |
| 任务重复执行 | 业务重复推进 | 去重与业务幂等 |
| 死信无人处理 | 长尾异常积压 | 死信台账与人工重放 |

## 18. 验收标准

### 18.1 功能验收

- 可完成入队、执行、重试、死信处理闭环。

### 18.2 数据验收

- 任务、尝试记录、死信结构完整。

### 18.3 技术验收

- 重试与退避策略生效。
- 任务可按业务键检索。

## 19. V1 边界

### 19.1 V1 必做

- 任务入队
- 重试
- 死信
- 执行日志

### 19.2 V1 暂不做

- 可视化任务编排
- 多数据中心任务调度编排

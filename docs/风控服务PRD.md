# 《风控服务详细 PRD》

## 1. 文档信息

| 项目 | 内容 |
|---|---|
| 文档名称 | 风控服务详细 PRD |
| 所属系统 | 交易平台后台 API 服务系统 |
| 模块名称 | 风控服务 / Risk Service |
| 版本 | V1.0 |
| 优先级 | P0 |
| 文档目标 | 明确同步风控校验、异步风险信号、规则、黑白名单、人工审核与风险事件边界 |

## 2. 模块背景与定位

### 2.1 模块背景

交易平台在下单、支付、履约、通知等环节都存在欺诈、刷单、异常渠道、资金风险等问题。如果没有独立风控服务，拦截规则会散落在多个业务服务中，难以维护。

### 2.2 一句话定位

风控服务是平台统一的风险检测与决策服务，负责风险规则、黑白名单、频控、评分、人工审核与风险事件输出。

### 2.3 模块价值

1. 在下单前拦截高风险请求。
2. 在交易进行中跟踪异常信号。
3. 为订单与渠道提供一致的风险决策口径。

## 3. 职责边界

### 3.1 风控服务负责什么

1. 同步预检风控。
2. 异步风险信号接收与聚合。
3. 黑白名单与频控规则。
4. 风险评分、拦截、人工审核。
5. 风险决策事件输出。

### 3.2 风控服务不负责什么

1. 不直接创建或关闭订单。
2. 不直接扣款或退款。
3. 不维护渠道主体或商品基础数据。
4. 不直接发送通知。

## 4. 设计原则

### 4.1 决策与执行分离原则

风控服务只输出风险决策，不直接执行订单关闭、账户冻结等动作。

### 4.2 同步与异步结合原则

下单前做同步预检，支付后、履约后、通知后可接收异步风险信号补充判定。

### 4.3 可解释原则

每次风险决策都应能说明命中规则、信号来源与建议动作。

## 5. 模块目标与成功标准

### 5.1 业务目标

- 在不明显影响主链路性能的前提下完成风险拦截。
- 支持黑白名单、频控、人工审核闭环。

### 5.2 技术目标

- 风险预检快速返回。
- 风险规则可配置、可审计。
- 风险事件可追溯。

### 5.3 成功标准

- 高风险请求可在下单前被识别。
- 规则命中与人工审核可追踪。
- 风险事件可按订单、渠道、IP 检索。

## 6. 角色与使用场景

| 角色 | 使用场景 |
|---|---|
| 平台风控 | 配置规则、处理审核单、维护黑白名单 |
| 订单服务 | 下单前请求同步风险预检 |
| 渠道服务 | 查询渠道风险状态 |
| 技术支持 | 排查风险拦截原因 |

## 7. 核心对象模型

| 对象 | 说明 |
|---|---|
| RiskRule | 风险规则 |
| RiskSignal | 风险信号 |
| RiskDecision | 风险决策 |
| BlackWhiteEntry | 黑白名单 |
| RiskReviewCase | 人工审核单 |

## 8. 状态模型

### 8.1 风险决策状态

- `PASS`
- `REVIEW`
- `REJECT`
- `DEGRADE`

### 8.2 审核单状态

- `PENDING`
- `APPROVED`
- `REJECTED`
- `CLOSED`

## 9. 核心流程设计

### 9.1 同步预检流程

1. 订单服务请求风控预检。
2. 风控服务拉取黑白名单、频控、规则集。
3. 输出 `PASS`、`REVIEW` 或 `REJECT`。

### 9.2 异步信号处理流程

1. 订单、支付、供应商、通知等服务输出异常信号。
2. 风控服务聚合信号并更新风险画像。
3. 必要时输出新的风险事件或生成审核单。

## 10. 功能需求明细

### 10.1 风险规则管理

- 支持频率类、黑白名单类、金额类、主体类规则。
- 支持启停、生效时间、优先级。

### 10.2 黑白名单管理

- 支持渠道、IP、手机号、主体、设备等维度。

### 10.3 人工审核

- 支持审核队列、审核意见、审核结论与留痕。

### 10.4 风险信号

- 支持接收支付异常、供应商异常、通知异常、渠道异常等信号。

## 11. 核心规则

1. 同步预检超时时必须有降级策略。
2. 白名单优先级高于普通规则。
3. 风险决策不直接改订单，只输出结果。

## 12. API 设计

### 12.1 后台管理 API

- `GET /admin/risk/rules`
- `POST /admin/risk/rules`
- `PUT /admin/risk/rules/{ruleId}`
- `GET /admin/risk/black-white-lists`
- `POST /admin/risk/black-white-lists`
- `GET /admin/risk/review-cases`
- `POST /admin/risk/review-cases/{caseId}/approve`
- `POST /admin/risk/review-cases/{caseId}/reject`

### 12.2 内部服务 API

- `POST /internal/risk/pre-check`
- `POST /internal/risk/order-events`
- `GET /internal/risk/channels/{channelId}/score`

| 表名 | 用途 |
|---|---|
| `risk_rules` | 风险规则 |
| `risk_black_white_list` | 黑白名单 |
| `risk_signals` | 风险信号 |
| `risk_decisions` | 风险决策 |
| `risk_review_cases` | 人工审核单 |

## 14. 跨服务协作

### 14.1 与订单服务

- 下单前输出预检结果。
- 订单异常时接收补充信号。

### 14.2 与渠道服务

- 读取渠道身份、IP、白黑名单上下文。

### 14.3 与支付、供应商、通知服务

- 接收异常信号作为风险画像输入。

## 15. 异常处理

### 15.1 风控服务超时

- 按规则配置走降级或拦截策略。

### 15.2 规则冲突

- 按优先级与白名单优先原则处理。

## 16. 权限、审计与非功能要求

### 16.1 权限要求

- 风控规则与黑白名单维护需严格授权。

### 16.2 非功能要求

- 同步预检接口平均响应小于 100ms。
- 审核与规则变更需审计。

## 17. 风险点与控制策略

| 风险 | 影响 | 控制策略 |
|---|---|---|
| 规则误杀 | 业务损失 | 审核与灰度启用 |
| 风控超时 | 下单性能受影响 | 降级策略 |
| 规则散落 | 风控口径不一致 | 统一风控服务 |

## 18. 验收标准

### 18.1 功能验收

- 可完成同步预检、黑白名单、规则管理、人工审核。

### 18.2 数据验收

- 决策、信号、审核单结构完整可查。

### 18.3 技术验收

- 同步预检稳定可调用。
- 风险决策具备可解释性。

## 19. V1 边界

### 19.1 V1 必做

- 同步预检
- 黑白名单
- 频控
- 人工审核

### 19.2 V1 暂不做

- 复杂机器学习评分
- 可视化风控编排平台

# 《结算与账务服务详细 PRD》

## 1. 文档定位

结算与账务服务负责 ISP 话费充值 V1 的渠道余额校验、余额扣款、退款冲正与基础价差记账，是资金事实的唯一归口。

## 2. 职责边界

### 2.1 负责内容

- 渠道余额账户。
- 平台账务账户。
- 余额充足校验。
- 下单成功后的余额扣款。
- 失败退款与冲正。
- 订单成功后的基础价差记账。
- 流水查询与审计。

### 2.2 不负责内容

- 在线支付。
- 冻结与解冻。
- 复杂分润。
- 结算单工作流。
- 完整财务对账闭环。

## 3. 账户模型

| 账户类型 | 说明 |
|---|---|
| 渠道可用余额账户 | 渠道下单时直接扣减 |
| 平台清算账户 | 接收订单扣款与退款冲正 |
| 平台统计收益账 | 记录基础价差收益结果 |

说明：

- V1 不做冻结余额账户。
- 统计收益账可为逻辑账，不要求形成复杂财务科目体系。

## 4. 核心流水动作

| 动作 | 触发时机 |
|---|---|
| `ORDER_DEBIT` | 订单创建后、提单前扣减渠道余额 |
| `ORDER_REFUND` | 订单失败或超时后退款 |
| `ORDER_SUCCESS_CONFIRM` | 订单成功后确认成本与基础价差 |

## 5. 核心规则

1. 余额校验通过后才允许创建可执行订单。
2. 余额校验与扣款必须以同一笔内部事务完成，避免“校验通过但扣款失败”的脏状态。
3. 下单成功必须立即扣减渠道余额。
4. 退款必须原路退回渠道余额。
5. 同一订单退款只能成功一次。
6. 所有资金动作都必须写流水。
7. 流水不可删除，只能冲正。

## 6. 记账口径

### 6.1 下单扣款

- 渠道余额减少 `salePrice`
- 平台清算账户增加 `salePrice`

### 6.2 订单成功

- 记录 `salePrice - purchasePrice` 的基础价差结果
- 成本与收益数据用于报表与差异核对

### 6.3 订单退款

- 平台清算账户减少 `salePrice`
- 渠道余额增加 `salePrice`
- 退款操作幂等键固定为 `orderNo + REFUND`

## 7. 接口设计

### 7.1 后台 API

- `GET /admin/accounts`
- `GET /admin/ledger-entries`
- `GET /admin/ledger-entries/:entryId`

### 7.2 内部 API

- `POST /internal/ledger/check-balance`
- `POST /internal/ledger/debit`
- `POST /internal/ledger/refund`
- `POST /internal/ledger/confirm-success`

## 8. 数据设计建议

- `ledger.accounts`
- `ledger.account_ledgers`
- `ledger.order_profit_snapshots`

## 9. 异常处理

- 余额不足：拒单，不创建订单。
- 重复扣款：幂等拦截。
- 退款失败：保持 `REFUNDING` 并由 Worker 自动重试。
- 成本价异常：进入差异清单，不阻塞订单成功。

## 10. 验收标准

1. 可按渠道校验余额并扣款。
2. 失败订单可自动退款并冲正。
3. 同一订单不会重复退款。
4. 基础价差可用于报表与对账。

## 11. V1 不做

- 在线支付收单。
- 冻结/解冻账户。
- 多级分润。
- 结算单与财务审批流。
